# RPA Snap Locate

[English](README.md) | [简体中文](README_zh.md)

轻量级 Windows 鼠标操作录制与回放工具。录一次，以后即可重复执行。

## 它能记录什么

- 每次按下、松开按键的时刻，以及两次操作之间的间隔；
- 准确记录快速双击，不会自作主张插入固定等待；
- 支持鼠标左键、右键和中键；
- 记录点击位置，适配常见屏幕缩放设置；
- 用 F3 标记区域，回放时自动截图；
- 每次回放前后自动截图，并报告操作是否明显超时；
- 也可选择只保存点击位置和顺序，不记录操作间隔。

## 快速开始

先安装依赖：

```bash
uv sync
```

启动录制：

```bash
uv run python main.py
```

只需要记住这一条主流程：

1. 按 F2 开始录制。
2. 正常完成鼠标操作，双击就按平时的速度双击。
3. 按 Ctrl+S 保存（正在录的话会先停再存）。
4. 按 F5 回放刚保存的文件。

按下 F2 即开始计时，因此第一次鼠标操作前的等待时间也会在回放时还原。先按 F2 停止再按 Ctrl+S 保存，效果相同。只有在想分多次录制、最后一次性保存时，才需要在中间按 F2 暂停；暂停期间不计时，再次按 F2 后继续。

也可以指定文件回放：

```bash
uv run python main.py run data/workflows/<文件名>.json
```

## 录制按键

| 按键 | 作用 |
| :--- | :--- |
| Ctrl+Delete | 清空尚未保存的操作记录 |
| Ctrl+S | 保存（正在录制则先停止再保存） |
| ESC | 取消进行中的框选 |
| F1 | 显示本帮助 |
| F2 | 开始录制；再次按下停止并保留，可继续追加 |
| F3 | 框选截图范围：按两次 F3 标记矩形区域，回放时自动截图保存；不改变点击定位方式 |
| F5 | 回放最近保存的文件 |

**使用框选截图：**

先按 F2 开始录制，然后：

1. 把鼠标放到区域的一个角，按 F3。
2. 移到对角，再按一次 F3。

该区域即作为截图事件加入时间线，回放时会在对应时刻自动截图。录制和点击定位不受影响（始终使用绝对坐标）。可标记多个区域，回放时每个区域截图保存到 `data/recordings/` 目录。

## 常见提示说明

- F3 提示无法使用：先按 F2 开始录制，再进行框选。
- 保存提示鼠标没有松开：按 Ctrl+Delete 清空，再重新录制一次。
- F5 提示有未保存内容：先按 Ctrl+S 保存，或者按 Ctrl+Delete 放弃。

## 回放结果

录制文件保存在：

```text
data/workflows/
```

每次回放的开始画面、结束画面和时间记录保存在：

```text
data/recordings/<录制文件名>/<回放时间>/
```

如果某次操作比预期超时较多，`replay_report.json` 会把本次回放标记为 `degraded`。这通常表示当时电脑繁忙、窗口切换较慢，或录制的操作本身间隔过短。degraded 不会中断回放，所有操作仍然执行。如果频繁出现，可尝试关闭后台程序或适当拉开操作间隔。

## 常用设置

编辑 `config/system.yaml`：

| 设置 | 默认值 | 用途 |
| :--- | :--- | :--- |
| `recorder.mode` | `timeline` | `timeline` 保留操作时间间隔；`legacy` 只保存点击位置和顺序 |
| `recorder.event_queue_limit` | `10000` | 一次录制最多暂存多少个鼠标事件 |
| `replay.start_delay_seconds` | `0` | 回放开始前等待几秒，方便切换窗口 |
| `replay.late_warning_ms` | `10` | 操作晚于计划多少毫秒时给出提醒 |

通常只需要调整 `start_delay_seconds`。例如设为 `2`，运行回放后会先等两秒。

完整配置项列表（屏幕、DPI、路径、队列上限等）请参考[架构文档中的配置章节](docs/ARCHITECTURE.md#16-配置项及真实生效状态)。

## Windows 权限

回放程序和目标软件必须使用相同权限：

- 目标软件普通运行时，本工具也普通运行；
- 目标软件以管理员身份运行时，本工具也要以管理员身份运行。

权限不同会导致 Windows 拒绝鼠标操作，程序会直接报错，不会假装执行成功。
