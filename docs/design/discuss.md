# 时间间隔定义

## 已确认规则

- 用户按 F2 开始录制时立即开始计时，不等到第一次鼠标操作才开始。
- 第一次鼠标操作或 F3 截图前的等待属于录制内容，回放时需要还原。
- 再次按 F2 停止录制后，停止期间不计时。
- 用户再次按 F2 继续录制时建立新的分段时间原点；该分段第一次操作前的等待同样需要还原。
- 多个录制分段合并时，只拼接各分段内记录的时间，不加入分段之间的暂停时间。

## 用户示例

用户按 F2，等待 5 秒后点击 A，再等待 2 秒点击 B。回放时先等待 5 秒点击 A，再等待 2 秒点击 B。

如果用户随后按 F2 停止，30 秒后再按 F2 继续，并在 3 秒后点击 C，回放不会等待停止期间的 30 秒，只会在上一分段结束后等待 3 秒再点击 C。

## 实现与兼容要求

- 单调时钟的原点在每次 F2 开始或继续录制时建立。
- `offset_ns` 仍表示事件相对所属录制分段起点的偏移，分段合并后转换成整条时间线上的绝对偏移。
- 旧版 v5 文件的首事件偏移为 `0`，新版首事件偏移可以大于等于 `0`；校验与回放必须同时兼容。
- 回放仍使用绝对截止时间，单次迟到不能改变后续事件原定时间。

## 验收标准

1. F2 后等待一段时间再点击，首事件偏移应等于这段等待时间，而不是 `0`。
2. 同一分段内后续事件间隔保持不变。
3. 分段之间的暂停时间不进入最终时间线。
4. 旧版首事件偏移为 `0` 的 v5 工作流仍可校验和回放。
