## 一、项目当前目标与边界

### 1.1 当前阶段核心目标

构建一个**纯视觉驱动的桌面自动化录制工具**。它通过快捷键（F2/F3）快速采集界面元素的坐标或截图区域，并持久化为标准化任务配置，为后续自动化执行提供精确的“操作蓝图”。

**当前只实现录制功能，不包含回放执行。**

### 1.2 录制能力边界

| 维度 | 能做到（✅） | 不涉及（❌） |
| :--- | :--- | :--- |
| **采集模式** | F2 快捷键录制单击坐标；F3 快捷键录制矩形截图区域（两个对角点） | 连续动作流录制、拖拽路径录制 |
| **坐标处理** | 自动获取当前鼠标位置、活动窗口，计算逻辑归一化坐标和DPI缩放比 | 智能元素识别（未来可扩展） |
| **截图存储** | 按F3框选的区域自动截取PNG模板图，并记录区域信息与偏移量 | 图像预处理、自动锚点选取 |
| **输出格式** | 生成标准 `workflow.json` 步骤序列，模板图存入 `templates/` | 执行回放、流程调度 |

## 二、整体架构（录制视角）

框架依然保持 **“感知-决策-执行”** 三层设计，但当前阶段仅激活感知层和录制决策层，执行层预留接口。

```text
[ 用户交互 ] 
     │  F2 / F3 热键
     ↓
┌─────────────────────────────────────────────┐
│           录制引擎 (Recorder Engine)         │
│  ┌──────────────┐  ┌──────────────────────┐ │
│  │ 感知模块     │  │  录制器 (Recorder)   │ │
│  │ (截屏/DPI/窗口)│  │  - F2: 记录点击坐标  │ │
│  │              │  │  - F3: 框选截图区域  │ │
│  └──────────────┘  └──────────────────────┘ │
│                     │                       │
│                     ↓                       │
│              workflow.json + templates/     │
└─────────────────────────────────────────────┘
```

- **感知模块**：仍负责截屏、获取鼠标/窗口信息、DPI 感知，为录制提供准确的物理和逻辑环境数据。
- **录制器**：响应热键，将原始数据加工为标准化的操作步骤，并写入配置文件。
- **执行模块与验证模块**：代码骨架保留，但当前版本不接入，等回放功能激活时再用。

## 三、快捷键录制详细设计

### 3.1 F2 — 录制单步点击坐标

**操作流程：**

1. 用户将鼠标移动到目标按钮/元素上方。
2. 按下键盘 **F2** 键。
3. 录制器捕获：
   - 当前鼠标的物理坐标 `(phys_x, phys_y)`。
   - 当前活动窗口的句柄和标题。
   - 当前系统的 DPI 缩放比例。
4. 计算逻辑归一化坐标（基于当前逻辑分辨率）：
   - `norm_x = phys_x / 逻辑屏幕宽度`
   - `norm_y = phys_y / 逻辑屏幕高度`
   （若启用窗口锚点模式，则计算相对于窗口左上角的归一化偏移）
5. 构造步骤对象：

   ```json
   {
     "action": "click",
     "method": "fixed",
     "norm_x": 0.35,
     "norm_y": 0.42,
     "window_title": "记事本",
     "dpi_scale": 1.5
   }
   ```

6. 追加到内存中的步骤列表。

**设计考量**：F2 只负责记录“点击”这个动作类型，默认左键单击。后续可通过配置文件扩展 action 类型（如双击、右键），但录制阶段统一为单击，简化交互。

### 3.2 F3 — 录制矩形截图区域（两个对角点）

**操作流程：**

1. 用户将鼠标移动到矩形区域的**第一个角**（例如左上角），按下 **F3** 键（第一次）。
   - 录制器进入“框选模式”，记录第一个点 `(x1, y1)`。
   - 可以在屏幕上绘制半透明矩形预览（可选，使用 `tkinter` 或直接绘制），方便用户确认区域。
2. 用户移动鼠标到**对角点**（例如右下角），再次按下 **F3** 键（第二次）。
   - 录制器记录第二个点 `(x2, y2)`。
3. 录制器自动计算标准矩形：
   - `left = min(x1, x2)`, `top = min(y1, y2)`
   - `right = max(x1, x2)`, `bottom = max(y1, y2)`
   - 宽度 `w = right - left`，高度 `h = bottom - top`
4. 调用感知模块，截取当前全屏，然后从全屏截图中裁剪出矩形区域的图像。
5. 将裁剪图像保存为 PNG 文件，存放到 `tasks/templates/` 目录，文件名使用时间戳或步骤索引，如 `step_003.png`。
6. **关键**：若该区域截图用于后续定位，还需要录制一个“目标点击点”在矩形区域内的相对位置。因此，录制器可要求用户再按一次 **F2** 在矩形内部点一下目标位置，或者直接以矩形中心作为默认目标点（可配置）。
   - 若使用 F2 补充目标点：计算目标点相对于矩形左上角的偏移量 `(offset_x, offset_y)`，写入配置。
   - 若使用默认中心：`offset_x = w/2, offset_y = h/2`。
7. 构造步骤对象：

   ```json
   {
     "action": "click",
     "method": "template",
     "template_path": "templates/step_003.png",
     "region": { "left": 100, "top": 200, "width": 80, "height": 30 },
     "offset_x": 40,
     "offset_y": 15,
     "window_title": "...",
     "dpi_scale": 1.5
   }
   ```

8. 追加到步骤列表，退出框选模式。

**交互细节**：

- 第一次按 F3 后，终端提示“框选模式：已记录第一个角，请移动鼠标到对角点，再次按 F3 确认”。
- 第二次按 F3 后，终端提示“框选完成，已截图保存。如需指定点击目标，请按 F2 在区域内点击；否则默认使用矩形中心。按 ESC 取消本次录制。”
- 若用户按 ESC，清除本次框选状态，不保存。

**兼容性**：F3 的两次按下需快速连续判断，可通过状态机实现（状态：IDLE → WAITING_SECOND_POINT → DONE）。

## 四、核心模块精简设计

### 4.1 感知模块 (`core/perception.py`)

保持不变，提供基础能力：

- `capture_screen()` → PIL Image
- `get_mouse_position()` → (x, y) 物理坐标
- `get_active_window()` → 窗口标题、句柄、位置
- `get_dpi_scale()` → 浮点数
- `get_logical_resolution()` → (width, height) 逻辑像素

### 4.2 录制器 (`engine/recorder.py`)

核心类：`Recorder`

- 属性：`steps: list`，`current_mode: str`（IDLE/BOX_SELECT）
- 方法：
  - `on_f2()`: 记录固定点击步骤
  - `on_f3_first()`: 开始框选，记录第一个点
  - `on_f3_second()`: 完成框选，截取区域，生成模板步骤
  - `on_f2_in_box()`: 在框选区域内记录精确点击偏移（可选）
  - `save_workflow(filepath)`: 将所有步骤序列化写入 `workflow.json`
  - `cancel_box_select()`: 取消框选

### 4.3 定位模块 (`core/locator.py`)

保留 `BaseLocator` 抽象接口，但在录制版本中不被调用，仅作为“未来执行时定位方式”的配置标识。录制时在步骤中记录 `method` 字段（`"fixed"` 或 `"template"`），为后续执行器的 locator 工厂提供依据。

### 4.4 执行模块 & 验证模块

模块文件保留为空壳或注释，待回放阶段开发。当前可完全忽略。

## 七、录制器交互流程（伪代码）

```python
def main():
    recorder = Recorder()
    register_hotkey('F2', recorder.on_f2)
    register_hotkey('F3', recorder.on_f3)   # 内部根据状态机区分第一次/第二次
    register_hotkey('ESC', recorder.cancel)
    print("录制器已启动：F2 录制点击，F3 框选截图区域，ESC 取消框选，Ctrl+S 保存工作流")
    wait_for_hotkeys()
```

### F3 状态机

- **IDLE** → 第一次 F3：记录 `(x1,y1)`，状态变为 `WAITING_SECOND`，提示用户。
- **WAITING_SECOND** → 第二次 F3：记录 `(x2,y2)`，计算矩形，截图，提示用户可选 F2 点目标或自动中心，状态变为 `WAITING_TARGET`。
- **WAITING_TARGET** → F2：记录目标点偏移，生成步骤，回到 `IDLE`。ESC 直接取消，回到 `IDLE`。
- 超时（如10秒无操作）自动取消。

## 八、后续扩展路径

1. **回放执行器**：启用 `runner.py`，读取 `workflow.json`，根据 `method` 字段实例化 `FixedLocator` 或 `TemplateLocator`，执行点击操作。
2. **自动锚点选取**：利用 `image_entropy.py` 在 F2 录制时自动识别高信息熵区域并截取模板，无需手动框选。
3. **多动作支持**：录制时允许通过组合键选择动作类型（如 Ctrl+F2 录制右键，Ctrl+F3 录制输入文本），丰富配置。
4. **可视化预览**：框选时绘制半透明矩形，提升用户体验。

当前交付的录制器已完整支持 F2 坐标点击和 F3 对角截图两种模式，生成的配置可直接作为未来自动化执行的唯一数据来源，结构清晰，拓展无忧。
